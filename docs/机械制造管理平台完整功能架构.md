# 机械制造管理平台完整功能架构

## 1. 建设目标

本平台面向机械制造企业，尤其适用于非标定制比例高、BOM 结构复杂、工艺路径多变、项目交付链条长的业务场景。系统目标不是简单叠加 ERP、MES、PLM、APS 功能，而是围绕同一条订单交付主线，打通设计、采购、生产、质检、发运、安装调试和成本核算。

平台建设需要始终围绕以下四个经营痛点展开：

1. 为了解决非标定制导致的频繁返工问题，系统必须建立从客户需求版本到设计版本、制造版本、工单版本的全链路版本控制。
2. 为了解决多层 BOM 和多工艺路线失控问题，系统必须让 BOM 直接驱动采购计划、工艺路线、齐套分析和生产执行。
3. 为了解决交期不准和插单冲击问题，系统必须具备有限产能排产、瓶颈工序负荷分析和缺料预警能力。
4. 为了解决单项订单赚不赚钱看不清的问题，系统必须实现按订单号或项目号归集料工费，并把返工、报废、外协、安装调试成本一起纳入核算。

## 2. 总体业务架构

建议平台按 8 个业务层次建设。

### 2.1 经营与项目层

解决痛点：接单阶段技术条件不清、交付承诺随意、项目过程无法穿透。

核心能力：

1. 客户需求管理
2. 商机与报价管理
3. 合同与销售订单管理
4. 项目立项与里程碑管理
5. 目标成本与目标毛利管理
6. 安装调试、验收、回款联动

关键设计：

1. 建立 `客户 -> 合同 -> 销售订单 -> 项目号 -> 交付里程碑` 的统一主线。
2. 对大型成套设备引入类似 Primavera P6 的里程碑逻辑，将设计、采购、加工、装配、发运、现场安装、SAT 串成一条计划链。
3. 在接单前冻结关键约束，包括配置选型、关键外购件交期、目标毛利、承诺交期。

### 2.2 PLM 与技术数据层

解决痛点：图纸版本混乱、EBOM 到制造端断层、变更无法评估影响。

核心能力：

1. CAD/PDM 集成
2. 图纸、3D 模型、技术文档版本管理
3. 物料主数据管理
4. EBOM、PBOM、MBOM 管理
5. ECN/ECR 变更控制
6. 标准件库、通用模块库、替代料库

关键设计：

1. 物料主数据必须至少包含编码、名称、规格、材质、单位、图号、版本、来源属性、自制/外购/外协属性。
2. 设计侧维护 EBOM，制造侧维护 PBOM 或 MBOM，并保留转换关系。
3. ECN 不能只记录审批结果，还必须生成影响清单，区分已采购、在制、待装配、已发运四类对象的处置策略。
4. 设计变更一旦生效，要自动推送到采购、库存、工艺、排产和成本模块。

### 2.3 工艺与制造工程层

解决痛点：工艺知识散落在个人经验里，非标产品工艺难复用，工位执行标准不统一。

核心能力：

1. 工艺模板管理
2. 工序路线管理
3. 工作中心与工位管理
4. 工装夹具与刀具管理
5. 标准工时与准备工时管理
6. 委外工序管理

关键设计：

1. 工艺模板要支持按产品族、设备类型、结构类型复用。
2. 每道工序至少绑定工作中心、标准工时、前置约束、检验点、报工规则和是否委外。
3. 工艺路线必须支持并行工序、返工工序、替代路线和跳站审批。
4. 工艺版本需受控，且必须与 MBOM、工单版本一一关联。

### 2.4 计划与 APS 层

解决痛点：人工计划粗放、插单影响不可评估、瓶颈资源长期过载。

核心能力：

1. 主生产计划 MPS
2. MRP 运算
3. 有限产能排产 APS
4. 瓶颈资源负荷分析
5. 齐套校验与缺料预警
6. 插单模拟与延期预测

关键设计：

1. 排产必须同时考虑订单优先级、交期、物料齐套、设备产能、工装占用、班次日历、外协周期。
2. 大型设备制造建议采用 `项目倒排 + 瓶颈工序有限产能 + 关键件齐套校验` 的混合排产策略。
3. 交期承诺不能只看理论工期，必须叠加采购前置期、委外前置期、质量放行时间和装配窗口。
4. 输出结果要能直接给计划员提供延误预警、缺料清单、负荷热力图和插单影响分析。

### 2.5 MES 执行层

解决痛点：现场透明度低、工位报工滞后、WIP 状态不清、追溯断裂。

核心能力：

1. 工单下达
2. 工位派工
3. 扫码报工
4. 在制品 WIP 管理
5. 委外协同
6. 条码或二维码追溯
7. 异常上报与 Andon

关键设计：

1. 报工不只记录完工数量，还要记录返工、报废、待料、停机、换型、异常原因。
2. 在制品需要按工序、工位、批次、序列号进行状态跟踪。
3. 现场执行支持派工单、工艺卡、检验卡、条码流转一体化。
4. MES 实际报工必须实时回写到订单进度、WIP 状态和成本模块。

### 2.6 SCM/WMS 资源保障层

解决痛点：缺料停工、呆滞料增加、供应商响应慢、仓储账实不一致。

核心能力：

1. 采购申请与采购订单
2. 供应商协同门户
3. 安全库存预警
4. 到货、质检、入库管理
5. 库位、批次、条码管理
6. 备料、发料、退料、补料管理
7. 项目物料池与齐套分析

关键设计：

1. MRP 结果要自动区分标准件、外协件、关键长周期件、项目专用件。
2. 缺料分类建议拆成五类：设计未释放、请购未下达、供应商延期、来料不合格、库存不可用。
3. 发料规则要支持按订单、按工单、按工序、按项目里程碑拉动。
4. 对于大型机械设备，建议建立项目物料池，防止跨项目挪料失控。

### 2.7 QMS 与 EAM 层

解决痛点：质量问题难追责、返工成本不透明、设备故障冲击交期。

核心能力：

1. IQC、IPQC、FQC、OQC 全流程质量管理
2. 不合格品 NCR 与 CAPA
3. 质量追溯与质量成本分析
4. 设备台账与设备履历
5. 点检、保养、校准、维修工单
6. 备件与预防性维护

关键设计：

1. 质量追溯必须能穿透到来料批次、图纸版本、工序参数、操作员、设备、检具和最终订单。
2. 返工返修应自动回写工时和材料损耗，纳入订单成本。
3. 关键设备要维护稼动率、停机原因、故障 MTBF、维修 MTTR 等指标。
4. 设备保养计划应与 APS 排产联动，避免关键设备在高峰期停机。

### 2.8 BI 与经营分析层

解决痛点：看板很多但经营洞察不足，交付风险发现太晚，项目毛利失真。

核心能力：

1. 订单或项目成本台账
2. 实时交付进度驾驶舱
3. 设备 OEE 与稼动率分析
4. 质量一次合格率与返工率分析
5. 采购交付表现与供应商评级
6. 订单利润复盘与偏差归因

关键设计：

1. 成本至少分解到材料、加工、装配、外协、质量、物流、安装调试七大块。
2. 经营看板不能只看结果，要能 drill-down 到订单、工序、物料、供应商、设备和责任部门。
3. 对大型项目建议建立穿透式查询，从项目看板直接下钻到里程碑、缺料、瓶颈工序、异常工单和回款状态。

## 3. 核心主数据对象

如果当前项目要继续往制造业平台演进，优先级最高的是统一核心对象。

### 3.1 五大核心对象

1. 订单
   说明：销售订单、制造订单、项目号、交付里程碑的统一入口。
2. 物料
   说明：所有标准件、自制件、外购件、外协件的主数据基础。
3. BOM
   说明：EBOM、PBOM、MBOM 多版本、多层级、多来源转换关系。
4. 工艺
   说明：工艺模板、工序路线、工作中心、工时标准、检验点。
5. 工单
   说明：MES 执行对象，承接派工、报工、检验、WIP 和成本。

### 3.2 建议补充的核心对象

1. 项目里程碑
2. 工作中心
3. 设备
4. 供应商
5. 质量单据
6. 委外工单
7. 成本归集记录
8. 安装调试任务

## 4. 端到端业务流程

建议平台的主流程按以下链路打通：

1. 销售接单
   生成客户需求版本、报价版本、目标交期和目标成本。
2. 设计输出
   形成图纸、EBOM、关键参数、技术说明和版本冻结点。
3. 工艺转化
   将 EBOM 转换为 PBOM/MBOM，生成工艺路线、工作中心、标准工时和检验点。
4. 计划运算
   依据订单、BOM、库存、在途、产能、工装和外协周期运行 MRP + APS。
5. 采购与备料
   自动形成采购申请、齐套分析、到货预警、安全库存预警。
6. 生产执行
   工位派工、扫码报工、工序过站、异常上报、委外协同。
7. 质量放行
   完成 IQC、IPQC、FQC、OQC，关闭 NCR/CAPA。
8. 发运与现场调试
   关联发货、安装、调试、SAT、验收和回款节点。
9. 成本复盘
   汇总材料、工时、外协、返工、质损、物流、现场服务成本，形成项目毛利分析。

## 5. 关键集成原则

### 5.1 数据流必须一条线打通

1. BOM 必须直接驱动采购需求、备料需求、工艺路线和工单展开。
2. MES 的工位报工、报废、返工、完工必须自动回写 ERP 成本模块。
3. ECN 生效后必须自动影响在采、在库、在制和在排计划对象。
4. 设备状态、停机、节拍、产出要自动进入 MES 和 OEE 看板。
5. 项目里程碑完成状态要自动联动制造进度和现场调试计划。

### 5.2 自动化优先

1. CAD 集成自动生成图纸索引与版本记录。
2. API 打通 PLM、ERP、MES、WMS、QMS、EAM、财务系统。
3. 对供应商到货回传、对账、签收等高频动作可引入 RPA。
4. AI 优先用于工时预测、交期预测、异常归因、工艺推荐和视觉质检。

## 6. 当前项目映射

当前项目不是从零开始，已经具备一部分制造平台雏形。

### 6.1 已具备的基础能力

1. 平台底座
   后端使用 FastAPI，前端使用 React + Vite，已经具备权限、部门、导入、仪表盘、AI 助手等基础能力。
2. PLM 雏形
   已有 `materials`、`bom_headers`、`bom_items`、`ecn_headers`、`ecn_impacts` 等基础模型。
3. 制造主对象雏形
   已有 `manufacturing_orders`、`order_items`、`production_steps` 等模型。
4. 工艺模板雏形
   已有 `process_templates`、`template_activities`、`template_activity_links`。
5. 进度与计划底座
   已有 P6 同步、WBS、活动、资源、Dashboard 等工程计划能力。
6. 前端入口雏形
   已有 `MaterialManagement`、`BOMStructure`、`ProcessTemplateConfig`、`ManufacturingCockpit` 页面。

### 6.2 当前最值得保留的方向

1. 继续把 P6 的项目里程碑能力保留为大型设备项目交付主线。
2. 将现有 BOM、工艺模板、制造订单雏形继续强化，不建议另起一套对象模型。
3. 把当前 AI 助手继续作为业务顾问入口，但后续应更多依赖真实业务数据回答，而不是固定话术。

### 6.3 当前主要缺口

1. 订单主线还不完整
   目前有制造订单模型，但销售订单、项目号、交付里程碑、安装调试链条还未完全串起来。
2. BOM 层级还偏基础
   当前模型已覆盖物料和基础 BOM，但还缺少 EBOM/PBOM/MBOM 的明确分层和转换规则。
3. 工艺到执行的闭环还未打通
   当前有工艺模板和生产步骤，但缺少标准工时、报工明细、WIP 状态流和工位级异常闭环。
4. 采购和库存主线缺失
   缺少 MRP、采购申请、齐套、库位、批次、发退料、项目物料池。
5. 成本与质量闭环不足
   缺少订单成本归集、返工返修成本、质损成本和供应商质量表现分析。
6. APS 尚未形成真正的排产能力
   当前更多是工程进度与模板逻辑，未形成基于设备产能、工时、齐套、瓶颈的有限产能排产。

## 7. 建议实施路径

为了避免系统功能堆砌，建议按三个阶段推进。

### 第一阶段：主数据与版本控制先打牢

目标：把订单、物料、BOM、工艺、工单五类核心对象统一起来。

重点事项：

1. 统一订单主线，补齐销售订单、项目号、制造订单、交付里程碑关系。
2. 扩展物料主数据，补齐物料来源、材质、图号、版本、替代料、自制外购属性。
3. 明确 BOM 分层，建立 EBOM 到 PBOM/MBOM 的转换关系。
4. 完善 ECN 影响评估，支持对在采、在制、已发料对象进行分类处置。
5. 把工艺模板和 BOM、工单绑定起来。

验证标准：

1. 任意一个订单可以穿透查到图纸版本、BOM 版本、工艺版本、工单版本。
2. 任意一个变更单可以查出具体影响了哪些采购单、工单、库存和项目里程碑。

### 第二阶段：执行闭环做实

目标：形成可落地的生产执行主链。

重点事项：

1. 建立工单下达、派工、扫码报工、WIP、检验过站流程。
2. 建立工作中心、设备、工位、班次日历和标准工时。
3. 建立发料、退料、补料和工序级齐套控制。
4. 建立 IQC、IPQC、FQC 与返工返修闭环。
5. 建立委外工序与供应商在线协同。

验证标准：

1. 订单进度可以穿透到具体工序、工位、操作员、设备和异常原因。
2. 返工、报废、待料、停机数据都能沉淀并可统计。

### 第三阶段：智能排产与经营分析

目标：把平台从记录工具升级为决策平台。

重点事项：

1. 建立有限产能 APS，输出负荷热力图、插单影响和延期预测。
2. 建立订单级成本台账，形成料工费与偏差分析。
3. 建立 OEE、质量、交付、供应商、项目毛利一体化看板。
4. 引入 AI 辅助工时预测、交期预测、视觉质检和工艺推荐。

验证标准：

1. 计划员可基于系统结果进行插单模拟和延期预警。
2. 财务和项目经理可按订单号实时看到料工费和毛利偏差。

## 8. 对当前项目的直接建议

如果要继续在当前项目上推进，我建议下一批开发任务不要分散，而是围绕一个明确切口展开。

最优先切口建议如下：

1. 先补齐“订单主线 + BOM 分层 + 工艺绑定”
   原因：这是后续采购、排产、报工、成本的共同基础。
2. 再补“工单执行 + 报工 + WIP + 质量过站”
   原因：这是从平台展示走向真实现场闭环的关键一步。
3. 最后补“MRP/齐套 + APS + 成本核算”
   原因：这些能力依赖前面对象和执行数据足够稳定。

如果只选一个最适合当前代码基础的短期开发主题，建议优先做：

`制造订单主线重构：销售订单/项目号/BOM/工艺模板/生产步骤一体化`

这个切口最容易与当前已有模型衔接，也最能为后续 SCM、MES、APS、BI 打基础。

## 9. AI 进阶方向

1. AI 辅助排产
   基于历史工时、设备负荷、缺料状态、插单规律给出重排建议。
2. AI 工艺助手
   根据产品族、结构特征、历史项目推荐工艺路线和标准工时。
3. AI 视觉质检
   用于焊缝、喷涂、外观缺陷、装配漏项识别。
4. AI 交期预警
   综合设计释放、供应商交付、瓶颈工序、返工率预测延期风险。
5. AI 经营复盘
   自动识别订单毛利偏差的主因，区分设计变更、材料涨价、外协超支、返工、低稼动率等因素。

## 10. 结论

机械制造平台的成败不在于模块数量，而在于是否围绕统一订单主线建立主数据、版本、执行、成本四个闭环。

对当前项目而言，正确方向不是继续零散加页面，而是把已经存在的 P6 项目能力、PLM/BOM 雏形、工艺模板能力、制造驾驶舱入口组织成真正可运行的制造业务骨架。
