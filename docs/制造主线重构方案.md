# 制造主线重构方案

## 1. 本轮已完成的收口

### 1.1 平台入口

- 前端默认入口保留为 `/manufacturing`
- 遗留工程计划、P6、WBS、日报、工程量等页面已从主路由移除
- 主导航已收缩为“制造主线 / 质量与协同 / 平台治理”

### 1.2 帮助中心

- 帮助中心文档已切换为机械制造语义
- 新增制造架构、主线流程、Excel 导入导出约定文档

### 1.3 后端主入口

- 后端 API 主注册已移除遗留工程计划相关默认路由
- 保留制造主线所需的 PLM、工艺、制造订单、采购交换、质量、设备、设施、权限等接口

## 2. 当前系统统一主线

建议后续所有需求都围绕以下对象链展开：

`销售订单/项目 -> 设计版本 -> BOM版本 -> 工艺版本 -> 制造订单 -> 工单/工序 -> 采购申请/采购单 -> 收货/IQC -> 报工/IPQC/FQC -> 发运/调试`

## 3. 当前仍需继续补齐的关键断点

### 3.1 订单主线对象不完整

当前制造订单还缺：

- 销售订单号
- 项目号
- 设计冻结版本
- 交付里程碑
- 成本归集主键

### 3.2 BOM 版本链未完成

当前还需要补：

- EBOM -> PBOM / MBOM 映射关系
- 派生版本与生效规则
- 工艺版本与 BOM 版本绑定

### 3.3 工艺模板仍带旧 work package 语义

需要把工艺模板升级为真正的制造工艺对象：

- 工序
- 工作中心
- 工位
- 标准工时
- 准备工时
- 检验点
- 委外工序

### 3.4 库存与采购口径仍偏静态

后续应改造成事务化台账：

- 订单预留
- 工单领料
- 批次 / 仓位
- 到货 / 退料 / 补料
- 供应商交期回传

## 4. 建议下一开发顺序

### 第一阶段：订单主线重构

目标：

- 销售订单 / 项目 / 制造订单打通
- 设计版本、BOM、工艺、工单建立主键关联

### 第二阶段：版本链与齐套闭环

目标：

- 建立 EBOM / PBOM / MBOM
- 打通 ECN 对在采、在制、在排的影响
- 建立订单级齐套分析

### 第三阶段：执行与成本闭环

目标：

- 工位报工、工序过站、质量记录自动回写
- 建立订单级料工费核算
- 形成交付风险与利润复盘看板

## 5. 约束原则

### 数据流优先

- BOM 必须直接驱动采购和工单
- 报工和质检必须直接回写订单状态
- ECN 必须影响采购、制造和计划

### 项目制兼容

针对大型设备项目，要保留里程碑管理能力，但不再以工程计划模块作为主入口。

### 自动化导向

优先使用：

- API 集成
- Excel 批量导入导出
- RPA 辅助对账
- AI 辅助排产、催交、质检与延期预警
