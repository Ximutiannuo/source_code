# 专项计划管理 - 已确认需求说明

## 一、数据与库

- **库**：在现有 **projectcontrols** 库内建表（不新建库）。表字符集随库（建议库为 utf8mb4_unicode_ci）。
- **模板说明**：`Weekly Plan (Look-Ahead).xlsm` 仅为**汇总表模板参考**，不是数据源；读不读均可，字段以库表为准。
- **表**：`ahead_plan`（事实表）。每行 = 一个作业在某一周的计划量。周周期为**周五至周四**；**date = 周结束日（周四）**，**planned_units = 该周计划量**；月计划量 = 当月各周 planned_units 之和。
  - **主数据**：id, type_of_plan, activity_id, resource_id, date, planned_units, remarks, user_defined_activity_name
  - **审计**：created_at, create_by, updated_at, updated_by
  - **审批**：reviewed_at, reviewed_by, approved_at, approved_by, comments, commented_by
- **关联**：activity_id → activity_summary；resource_id 通过 activity_summary.work_package 关联 rsc_defines.work_package 取 rsc_defines.resource_id；scope 用 activity_summary.scope。

## 二、业务流程

- 分包在**平台**编制计划，工程师在**平台**审核、批准、提意见。
- **20 日**：提醒开始编制下月计划（站内+邮件，具体提醒对象下一步再定，**暂不实现**）。
- **25 日**：下月计划须编制、审核、批准完成；**25 日后**仅管理员可变更计划。

## 三、作业池与权限

- **作业池**：直接使用 **activity_summary**。
- **权限**：分包只能选**自己 scope** 的作业，与当前已在执行的权限管理一致。

## 四、计划类型与表设计

- **月滚动 / 三月滚动**：同一套表，用 **type_of_plan** 区分（如 `monthly` / `three_month`）。

## 五、审批/签字

- 需**记录时间与人**：reviewed_at, reviewed_by, approved_at, approved_by, comments, commented_by。
- 当前**一个施工队一个账号**，先按单账号处理；后续增加账号再改流程。

## 六、导入导出

- **导入**：支持从 **xlsx** 导入月计划到 `ahead_plan`（按周落事实表行）。
- **导出**：支持按附图模板导出 xlsx；模板中多数展示字段从 **activity_summary**（含 scope）及 **rsc_defines**（经 work_package 关联）带出。

## 七、前端与菜单

- **菜单**：「计划管理」下：原「计划管理」改为 **总体计划管理**，新增 **专项计划管理**。
- **专项计划页**：参考 activities-advanced；默认不载入作业，用户通过 **作业 ID 加载** 或从 **作业池选取** 到本月计划；右侧为**表单**（附图式表格），不用甘特图。
