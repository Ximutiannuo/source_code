# P6作业清单实时/周期性更新系统设计

## 一、系统概述

### 1.1 原有流程（Excel导出方式）
```
P6 Export → Excel文件 → PowerQuery → Activity(C_P6) → 算法处理 → Activity List
```

### 1.2 新流程（API方式）
```
P6 REST API → 临时表 → 数据转换 → Activity表 → 算法计算 → Activity List
```

## 二、数据源分析

### 2.1 P6 API数据源
1. **Activity（作业）**
   - 基本信息：Id, Name, Type, StatusCode
   - 日期信息：StartDate, FinishDate, PlannedStartDate, PlannedFinishDate, ActualStartDate, ActualFinishDate
   - 基线信息：Baseline1StartDate, Baseline1FinishDate
   - WBS信息：WBSCode, WBSPath (long)
   - 其他：IsCritical, IsLongestPath, Duration等

2. **ActivityCodeAssignment（作业代码分配）**
   - 用于获取：Scope, Discipline, Work Package, Contract Phase等分类信息
   - 通过ActivityCodeTypeName区分不同的代码类型

### 2.2 本地数据库数据源
1. **VolumeControl（工程量控制）**
   - keyqty (estimated_total) - 用于计算人工时

2. **RSCDefine（资源定义）**
   - norms_mp (unit_manhour) - 单位人工时
   - uom - 单位

3. **MPDB / VFACTDB（日报数据）**
   - 实际进度数据

### 3.3 更新策略层（Update Strategy Layer）

#### 3.3.1 全量更新（Full Sync）
- **适用场景**：首次导入、定期全量刷新
- **流程**：
  1. 清空Activity表（或标记为is_active=False）
  2. 从P6获取所有作业
  3. 转换并插入
  4. 计算人工时和权重
  5. 更新Activity List

#### 3.3.2 增量更新（Incremental Sync）
- **适用场景**：日常更新、新增作业
- **流程**：
  1. 从P6获取所有作业（或使用LastUpdateDate过滤）
  2. 对比现有数据（通过act_id）
  3. 插入新作业，更新已存在的作业
  4. 重新计算受影响作业的人工时和权重

#### 3.3.3 差异更新（Delta Sync）
- **适用场景**：只同步有变化的作业
- **流程**：
  1. 从P6获取作业的LastUpdateDate
  2. 只获取LastUpdateDate > 上次同步时间的作业
  3. 更新这些作业
  4. 重新计算受影响作业的人工时和权重

## 六、更新频率建议

1. **实时更新**：用户手动触发（通过前端按钮）
2. **周期性更新**：
   - 每日凌晨2:00（全量同步）
   - 每小时（增量同步，只同步最近1小时更新的）
3. **事件驱动更新**：P6数据变更时触发（需要P6支持webhook）

## 七、性能优化

1. **批量处理**：批量插入、批量计算
2. **增量更新**：只处理有变化的作业
3. **异步处理**：使用后台任务处理大量数据
4. **缓存机制**：缓存RSCDefine、VolumeControl等基础数据
