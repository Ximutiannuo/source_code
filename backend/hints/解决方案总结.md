# numpy/pandas 导入阻塞问题 - 解决方案总结

## 问题确认

✅ **已确认**：numpy 导入阻塞（不是 pandas）
- 诊断脚本在导入 numpy 时卡住
- pandas 依赖 numpy，所以也会阻塞

✅ **已确认**：Visual C++ Redistributable 已安装
- 系统已安装版本：14.50.35710.0（足够新）
- 不需要重新安装

## 可能的原因

既然 Visual C++ Redistributable 已安装且版本足够新，问题可能是：

1. **numpy 安装损坏**
   - 二进制文件可能损坏
   - 需要重新安装

2. **numpy 版本不兼容**
   - 当前安装的 numpy 版本可能与 Python 3.12.10 不完全兼容
   - 需要安装兼容的版本

3. **系统权限问题**
   - numpy 的 DLL 文件可能无法正常加载
   - 检查文件权限

4. **其他系统依赖缺失**
   - 可能缺少其他运行时库

## 解决方案（按优先级）

### 方案 1：重新安装 numpy 和 pandas（推荐）

```powershell
# 1. 卸载现有版本
pip uninstall pandas numpy -y

# 2. 清理 pip 缓存
pip cache purge

# 3. 重新安装（指定兼容的版本）
pip install numpy==1.26.4
pip install pandas==2.2.2

# 4. 测试导入
python -c "import numpy; print('numpy OK'); import pandas; print('pandas OK')"
```

### 方案 2：使用预编译的 wheel 文件

如果方案 1 不行，尝试从官方源安装：

```powershell
# 卸载
pip uninstall pandas numpy -y

# 从官方 PyPI 安装（不使用缓存）
pip install --no-cache-dir numpy
pip install --no-cache-dir pandas
```

### 方案 3：检查 numpy 安装位置和权限

```powershell
# 检查 numpy 安装位置
python -c "import numpy; print(numpy.__file__)" 2>&1

# 如果上面的命令也卡住，直接检查文件
# 在虚拟环境中：myenv\Lib\site-packages\numpy
```

### 方案 4：使用延迟导入（临时方案）

如果以上方案都不行，可以暂时使用延迟导入：

- 在 `import_api.py` 中将 `import pandas as pd` 改为函数内部导入
- 这样启动时不会阻塞，只在需要时导入

## 诊断命令

运行以下命令进一步诊断：

```powershell
# 1. 检查 numpy 和 pandas 版本
pip list | findstr -i "numpy pandas"

# 2. 检查安装位置
python -c "import sys; print(sys.path)"

# 3. 尝试单独导入 numpy（带超时）
python check_pandas_installation.py
```
