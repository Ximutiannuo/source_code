# 数据库用户管理运维指南

## 概述

本文档介绍如何使用数据库用户管理工具来管理 MySQL 用户密码、权限，并同步更新 Vault 中的密码。

## 工具位置

`backend/scripts/manage_db_users.py`

## 功能

1. **更新 MySQL 用户密码** - 修改数据库中的用户密码
2. **同步 Vault 密码** - 同步更新 Vault 中存储的密码
3. **更新用户权限** - 授予用户访问数据库的权限
4. **验证连接** - 测试用户连接是否正常

## 使用方法

### 基本语法

```powershell
cd backend
python scripts/manage_db_users.py <action> [选项]
```

### 操作类型

- `update-password` - 更新 MySQL 和 Vault 中的密码
- `update-permissions` - 更新用户权限
- `sync-vault` - 仅同步 Vault 中的密码
- `verify` - 验证用户连接
- `all` - 执行所有操作（更新密码、权限、验证）

### 常用选项

- `--password <密码>` - 指定新密码（用于 update-password 和 sync-vault）
- `--role <角色名>` - 指定要处理的角色（可选，不指定则处理所有角色）
- `--skip-vault` - 跳过 Vault 更新
- `--skip-mysql` - 跳过 MySQL 更新

## 使用示例

### 1. 更新所有角色的密码

```powershell
python scripts/manage_db_users.py update-password --password "新密码"
```

这会：
- 更新所有角色用户在 MySQL 中的密码
- 同步更新 Vault 中的密码
- 刷新 MySQL 权限

### 2. 更新特定角色的密码

```powershell
python scripts/manage_db_users.py update-password --password "新密码" --role planning_manager
```

### 3. 仅更新 MySQL 密码（不更新 Vault）

```powershell
python scripts/manage_db_users.py update-password --password "新密码" --skip-vault
```

### 4. 仅更新 Vault 密码（不更新 MySQL）

```powershell
python scripts/manage_db_users.py sync-vault --password "新密码" --skip-mysql
```

### 5. 更新用户权限

```powershell
python scripts/manage_db_users.py update-permissions
```

这会授予所有角色用户访问 `projectcontrols` 和 `proecomcontrol` 数据库的权限。

### 6. 验证连接

```powershell
python scripts/manage_db_users.py verify
```

这会测试所有角色用户的数据库连接是否正常。

### 7. 完整操作（推荐）

```powershell
python scripts/manage_db_users.py all --password "新密码"
```

这会：
1. 更新所有角色的 MySQL 密码
2. 同步更新 Vault 中的密码
3. 更新用户权限
4. 验证连接

## 角色列表

- `role_planning_manager` (PLANNING_MANAGER) - 计划经理
- `role_system_admin` (SYSTEM_ADMIN) - 系统管理员
- `role_planning_supervisor` (PLANNING_SUPERVISOR) - 计划主管
- `role_planner` (PLANNER) - Planner

## 环境配置

工具会自动从 `.env` 文件读取配置：

- `DATABASE_URL` - MySQL 连接信息（用于获取 root 账号和数据库地址）
- `VAULT_ADDR` - Vault 服务器地址（默认: http://127.0.0.1:8200）
- `VAULT_TOKEN` - Vault 认证令牌

## 常见场景

### 场景 1: 定期更换密码

```powershell
# 1. 生成新密码（建议使用强密码）
$newPassword = "新强密码"

# 2. 更新所有角色的密码
python scripts/manage_db_users.py all --password $newPassword

# 3. 重启应用服务以清除连接池缓存
```

### 场景 2: 修复权限问题

如果用户无法访问数据库，可能是权限问题：

```powershell
# 1. 更新权限
python scripts/manage_db_users.py update-permissions

# 2. 验证连接
python scripts/manage_db_users.py verify

# 3. 如果验证失败，检查日志文件
```

### 场景 3: 同步 Vault 和 MySQL 密码

如果 Vault 和 MySQL 中的密码不一致：

```powershell
# 1. 确认正确的密码
# 2. 同步更新
python scripts/manage_db_users.py all --password "正确密码"
```

### 场景 4: 添加新角色用户

如果需要添加新的角色用户：

1. 在 MySQL 中创建用户（使用工具或手动）
2. 在 Vault 中添加密码
3. 授予权限：
   ```powershell
   python scripts/manage_db_users.py update-permissions --role 新角色名
   ```

## 权限说明

工具会授予以下权限：

- **projectcontrols 数据库**:
  - SELECT, INSERT, UPDATE, DELETE
  - CREATE, ALTER, INDEX, DROP

- **proecomcontrol 数据库**（如果存在）:
  - SELECT, INSERT, UPDATE, DELETE

## 日志文件

所有操作都会记录到日志文件：`backend/db_users_management.log`

日志包含：
- 操作时间
- 操作详情
- 成功/失败信息
- 错误堆栈跟踪

## 故障排查

### 问题 1: MySQL 连接失败

**错误**: `MySQL 连接失败`

**解决方案**:
1. 检查 `.env` 文件中的 `DATABASE_URL` 配置
2. 确认 MySQL 服务正在运行
3. 确认 root 密码正确
4. 检查网络连接

### 问题 2: Vault 连接失败

**错误**: `Vault 连接失败` 或 `Vault 认证失败`

**解决方案**:
1. 检查 Vault 服务是否运行
2. 确认 `VAULT_ADDR` 和 `VAULT_TOKEN` 环境变量正确
3. 验证 Token 是否有效且未过期

### 问题 3: 权限更新失败

**错误**: `权限授予失败`

**解决方案**:
1. 确认 root 用户有 GRANT 权限
2. 检查数据库是否存在
3. 查看日志文件获取详细错误信息

### 问题 4: 连接验证失败

**错误**: `连接验证失败`

**可能原因**:
1. 密码不正确
2. 用户不存在
3. 权限不足
4. 数据库地址不正确

**解决方案**:
1. 重新运行 `update-password` 确保密码正确
2. 运行 `update-permissions` 确保权限正确
3. 检查日志文件查看详细错误

## 安全建议

1. **密码管理**:
   - 使用强密码（至少 12 位，包含大小写字母、数字、特殊字符）
   - 定期更换密码
   - 不要在命令行中直接输入密码（使用环境变量或配置文件）

2. **权限最小化**:
   - 只授予必要的权限
   - 定期审查用户权限

3. **日志管理**:
   - 日志文件可能包含敏感信息，请妥善保管
   - 定期清理旧日志文件

4. **Vault Token**:
   - 不要将 Token 提交到代码仓库
   - 使用环境变量或安全的配置管理工具

## 注意事项

1. **连接池缓存**: 更新密码后，需要重启应用服务以清除连接池缓存

2. **数据库地址**: 工具会自动从 `.env` 文件读取数据库地址，确保配置正确

3. **批量操作**: 使用 `all` 操作时，会按顺序执行所有操作，请确保每一步都成功

4. **角色过滤**: 使用 `--role` 选项时，支持部分匹配（不区分大小写）

## 重要提示：数据库地址配置

**如果数据库在远程服务器，必须确保应用使用正确的数据库地址！**

应用会从 `.env` 文件中的 `DATABASE_URL` 自动解析数据库地址。如果 `DATABASE_URL` 是：
```
DATABASE_URL=mysql+pymysql://root:password@10.78.44.17:3306/projectcontrols?charset=utf8mb4
```

应用会自动使用 `10.78.44.17` 作为数据库地址。

**如果应用仍然连接 `localhost`，请检查：**
1. `.env` 文件中的 `DATABASE_URL` 是否正确
2. 应用是否已重启（配置更改需要重启生效）
3. 连接池缓存是否已清除
