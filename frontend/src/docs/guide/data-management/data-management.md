# 数据管理模块指南

数据管理模块用于**查看、维护、导入导出**人力数据（MPDB）、工程量数据（VFACTDB）、工程量清单（Volume Control）、作业清单，以及**工效分析**。本文档说明各页面的用途、筛选、操作与权限。

---

## 一、模块总览

| 页面 | 菜单位置 | 数据来源 | 用途 |
|------|----------|----------|------|
| **作业清单** | 数据管理 → 作业清单 | activity_summary（P6 同步） | 作业列表查询、导出；确认完成、重新打开、批量关闭 |
| **工程量清单** | 数据管理 → 工程量清单 | VolumeControl*（施工/验收/ABD/OVR） | 工程量控制表维护、导入导出、汇总统计 |
| **人力数据** | 数据管理 → 人力数据 | MPDB | 人力投入记录查看、导入导出 |
| **工程量数据** | 数据管理 → 工程量数据 | VFACTDB | 工程量完成记录查看、导入导出 |
| **工效分析** | 数据管理 → 工效分析 | productivity_cache | 工效趋势、对比分析 |

---

## 二、作业清单

### 2.1 入口与用途

* **入口**：数据管理 → **作业清单**
* **数据来源**：activity_summary，与 P6 同步（约 5 分钟）。
* **用途**：按筛选条件查询作业列表；导出 Excel；确认完成、重新打开、批量关闭（需权限）。

### 2.2 与总体计划管理的区别

| 项目 | 作业清单 | 总体计划管理 |
|------|----------|--------------|
| 路径 | 数据管理 → 作业清单 | 计划管理 → 总体计划管理 |
| 展示 | 表格列表 | 甘特图 + 作业清单 + 详情面板 |
| 功能 | 查询、导出、确认完成/重新打开/批量关闭 | 甘特图、分组、栏位、视图、详情、确认完成等 |
| 适用 | 快速查作业、批量操作 | 计划浏览、进度跟踪、详情编辑 |

### 2.3 筛选与导出

* 支持全局筛选器（分包商、子项目、主项、子项、装置等）。
* 支持本地筛选（block、search）。
* 导出：按当前筛选条件导出 Excel。
* 确认完成、重新打开、批量关闭：需 `daily_report:create`，详见《总体计划管理》。

## 三、工程量清单

### 3.1 入口与用途

* **入口**：数据管理 → **工程量清单**
* **数据来源**：VolumeControlQuantity、VolumeControlInspection、VolumeControlABD、VolumeControlOVR 等。
* **用途**：维护施工工程量、验收工程量、ABD 工程量、OVR 工程量；查看总量、到货、工作面、施工完成、验收、竣工、收款；汇总统计；导入导出。

### 3.2 工程量类型

| 类型 | 说明 | 权限 |
|------|------|------|
| **施工工程量** | construction_completed 等 | construction_volume:read/update |
| **验收工程量** | rfi_completed_a/b/c 等 | acceptance_volume:read/update |
| **ABD 工程量** | abd 相关 | abd_volume:read/update |
| **OVR 工程量** | ovr 相关 | ovr_volume:read/update |

### 3.3 筛选与操作

* 支持全局筛选器；本地筛选（block、search）。
* 汇总统计：可按工作包、主项、子项目、装置、子项、开车阶段、区块分组。
* 导入/导出：Excel 模板；导入时按 activity_id 匹配，更新对应字段。
* 施工部责任人、各工程量责任人：可在详情中维护。

### 3.4 与日报的关系

* **施工工程量**：由**工程量日报（VFACTDB）**汇总自动更新 construction_completed，一般不可手动修改。
* 验收、ABD、OVR 等可按需维护；验收日报与工程量清单当前不联动（见《验收日报》）。

## 四、人力数据（MPDB）

### 4.1 入口与用途

* **入口**：数据管理 → **人力数据**
* **数据来源**：MPDB（人力日报表）。
* **用途**：查看人力投入记录（Date、Activity ID、Scope、Type of Manpower、Manpower、Machinery 等）；导入导出；新增、编辑、删除（需权限）。

### 4.2 与人力日报的关系

* **人力日报**：日报填报 → 人力日报，用于**填报**当日人力；面向分包商/计划管理员。
* **人力数据**：数据管理 → 人力数据，用于**查看、维护、导入导出**已填报的记录；面向计划部/管理人员。

### 4.3 筛选与操作

* 支持全局筛选器；本地筛选（Date、Activity ID、Scope 等）。
* 栏位设置：可勾选显示的列。
* 导入：支持 Excel 批量导入。
* 导出：按当前筛选条件导出。
* 无效记录：若作业在 P6 中不存在或 Scope 不匹配，会标记为无效（P6不存在/Scope不匹配）。

## 五、工程量数据（VFACTDB）

### 5.1 入口与用途

* **入口**：数据管理 → **工程量数据**
* **数据来源**：VFACTDB（工程量日报表）。
* **用途**：查看工程量完成记录（Date、Activity ID、Scope、Work Step、Achieved 等）；导入导出；新增、编辑、删除、按周分配、批量调整（需权限）。

### 5.2 与工程量日报的关系

* **工程量日报**：日报填报 → 工程量日报，用于**填报**昨日工程量；面向分包商/计划管理员。
* **工程量数据**：数据管理 → 工程量数据，用于**查看、维护、导入导出**已填报的记录；面向计划部/管理人员。

### 5.3 筛选与操作

* 支持全局筛选器；本地筛选。
* 栏位设置：可勾选显示的列。
* 导入/导出：支持 Excel；可按周分配、批量调整（计划经理/计划管理员/系统管理员）。
* 无效记录：若作业在 P6 中不存在或 Scope 不匹配，会标记为无效。
* 施工完成量（construction_completed）由 VFACTDB 汇总至 VolumeControlQuantity，自动更新。

### 5.4 特殊操作

* **按周分配**：将某日完成量分配到各周（周五～周四）。
* **批量调整**：批量修改完成量。
* **删除/新增**：仅计划经理、计划管理员、系统管理员可操作。

## 六、工效分析

### 6.1 入口与用途

* **入口**：数据管理 → **工效分析**
* **数据来源**：productivity_cache、productivity_cache_wp。
* **用途**：查看工效趋势、按队伍/子项目/装置/主项分组对比；辅助分析劳动效率与异常。

### 6.2 筛选与维度

* 支持全局筛选器。
* 分组维度：按队伍（scope）、子项目、装置、主项。
* 时间范围：可自定义；支持累计工效与周期工效。
* 工程量类型：混凝土、钢结构、焊接等。

### 6.3 工效指标说明

* **工效**：完成量 / 投入工时（或按工作包规范折算）。
* **周期工效**：某时间段内完成量 / 该时段投入工时。
* **累计工效**：截止某日累计完成量 / 累计投入工时。
* **相对标准工效**：与预算或定额的对比；用于预警异常。
* 计算公式详见《计算逻辑说明》。

## 七、数据恢复控制台

* **入口**：数据管理 → 数据恢复控制台（或 系统管理 → 系统备份与恢复）
* **用途**：系统备份与恢复；仅系统管理员/超级管理员可访问。
* 详见系统管理相关说明。

## 八、权限速查

| 页面 | 查看 | 修改/导入/导出 |
|------|------|----------------|
| 作业清单 | planning:read | 确认完成/重新打开/批量关闭需 daily_report:create |
| 工程量清单 | construction_volume/acceptance_volume/abd_volume/ovr_volume:read | 对应 *_volume:update |
| 人力数据 | daily_report:read | daily_report:create/update/delete |
| 工程量数据 | daily_report:read | daily_report:create/update/delete；特殊操作需计划经理等 |
| 工效分析 | 无单独权限 | 只读，依赖其他数据权限 |
