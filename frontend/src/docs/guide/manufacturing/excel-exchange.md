# Excel 导入导出约定

## 一、适用范围

当前平台要求制造数据逐步支持 Excel 导入导出，重点覆盖：

- 制造订单
- 采购申请 / 采购请求
- 物料主数据
- BOM 明细
- 工序执行与进度数据

## 二、导出原则

- 导出的字段命名要与页面展示尽量一致
- 编码字段优先保留原始主键或业务编码
- 状态字段优先导出标准状态值，同时兼容中文展示
- 需要区分头表与明细表时，建议使用多 Sheet

## 三、导入原则

### 1. 支持覆盖更新

如果 Excel 中的订单号或请购单号已存在，系统应优先执行覆盖更新，而不是简单报“已存在”。

### 2. 优先按业务唯一键匹配

- 制造订单按 `order_number`
- 采购请求按 `request_no`
- 物料按 `material_code`
- BOM 按 `product_code + version`

### 3. 进度类字段必须有回写规则

例如导入工序进度时，需要明确：

- 是回写工序完工数量
- 还是直接回写订单状态
- 是否同时触发工单状态重算

否则页面会出现“Excel 已导入，但进度仍是 0”的问题。

## 四、建议字段口径

### 制造订单

- 订单号
- 客户
- 产品名称
- BOM 编码 / 版本
- 工艺模板
- 数量
- 交期
- 优先级
- 状态
- 备注

### 采购请求

- 请购单号
- 来源范围
- 来源订单号
- 紧急程度
- 状态
- 申请人
- 备注

### 工序执行

- 订单号
- 工序编码
- 工序名称
- 目标数量
- 已完工数量
- 计划工时
- 准备工时
- 工位 / 设备
- 状态

## 五、后续增强建议

- 为每个导出模板增加“字段说明 Sheet”
- 导入结果返回新增数、更新数、失败数和失败原因
- 对订单、采购、工序提供“预览导入”模式
- 对关键导入增加覆盖策略提示：仅空值覆盖 / 全量覆盖 / 明细替换
