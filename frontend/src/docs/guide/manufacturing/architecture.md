# 平台功能架构

## 一、核心业务域

### 1. PLM 技术源头

- 物料主数据
- 图纸与版本
- EBOM / PBOM / MBOM
- ECN / ECR 变更控制

目标：让设计输出成为采购和制造的可信单一来源。

### 2. 工艺与制造工程

- 工艺模板
- 工序路线
- 工作中心 / 工位
- 标准工时 / 准备工时
- 检验点与委外工序

目标：把工艺经验沉淀成可复用、可排产、可报工的制造规则。

### 3. SCM / WMS 资源保障

- MRP 运算
- 齐套分析
- 采购申请 / 采购执行
- 到货与来料状态
- 在制品和预留控制

目标：让 BOM 需求、库存可用量、采购动作自动联动。

### 4. MES 执行闭环

- 工单下达
- 工位报工
- 工序过站
- WIP 跟踪
- 异常上报

目标：形成订单到工序的透明执行链。

### 5. QMS / EAM 保障

- IQC / IPQC / FQC / OQC
- NCR / CAPA
- 设备台账
- 保养计划
- 稼动率 / OEE

目标：把质量和设备从“事后统计”变成“事中控制”。

### 6. BI 经营分析

- 订单交付进度
- 缺料风险
- 质量损失
- 设备效率
- 单项订单成本与利润复盘

目标：支持穿透式查询，而不是只看结果数字。

## 二、当前系统入口结构

系统已经统一为三个入口分区：

1. 制造主线
2. 质量与协同
3. 平台治理

原遗留工程模块已经从默认入口移除，不再作为主业务路径的一部分。

## 三、下一阶段重点

### 1. 订单主线统一

需要逐步补齐：

- 销售订单
- 项目号
- 设计冻结版本
- 制造订单
- 采购申请
- 工单 / 工序
- 质量单据

### 2. 版本链打通

需要建立：

- EBOM -> PBOM / MBOM 转换关系
- 工艺版本 -> 工单执行版本关系
- ECN -> 在采 / 在制 / 在排对象影响分析

### 3. 库存口径事务化

需要从静态库存字段逐步升级到：

- 订单级预留
- 工单级发料
- 批次 / 仓位
- 到货 / 退料 / 补料台账

这样采购建议和齐套分析才能真实可信。
