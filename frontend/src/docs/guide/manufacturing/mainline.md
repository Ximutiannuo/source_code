# 设计-采购-制造主线

## 一、主线目标

当企业需要生产一个零件或一台设备时，系统应该天然支持下面这条业务链，而不是让各模块各自记录、彼此脱节：

1. 设计输出图纸和 EBOM
2. 工艺把 EBOM 转换成制造可执行的 PBOM / MBOM
3. 采购根据 BOM、库存、交期生成请购与催交动作
4. 制造依据工艺路线展开工单和工序
5. 质量在工序节点和完工节点完成放行
6. 设备数据回写稼动率、停机和瓶颈分析

## 二、关键对象关系

推荐主对象关系如下：

`销售订单/项目 -> 设计版本 -> BOM版本 -> 工艺版本 -> 制造订单 -> 工单/工序 -> 采购申请/采购单 -> 质检 -> 交付`

## 三、各环节应看到什么

### 1. 设计端

需要看到：

- 当前订单对应的图纸版本
- EBOM 与替代料
- 变更影响范围
- 哪些采购单和工单会受 ECN 影响

### 2. 采购端

需要看到：

- 按订单的缺料清单
- 按物料的净需求
- 安全库存风险
- 供应商承诺交期
- 哪些工单被该物料卡住

### 3. 制造端

需要看到：

- 当前订单工序进度
- 工位报工状态
- WIP 所在工序
- 待检、返工、报废数量
- 设备稼动率和停机原因

## 四、当前系统已收口的方向

当前平台已经完成以下收口：

- 前端主导航以制造主线为中心
- 默认入口改为制造驾驶舱
- 遗留工程计划模块已从默认路由和导航移除
- 帮助中心改为机械制造业务文档

## 五、后续建议

为了让主线真正闭环，建议按下面顺序推进：

### 第一阶段

- 补齐销售订单 / 项目 / 制造订单关系
- 建立 BOM 与工艺绑定
- 让工艺模板真正脱离 work package 语义

### 第二阶段

- 建立订单级齐套分析
- 打通采购申请、到货、来料检验
- 建立工位报工、IPQC、FQC 闭环

### 第三阶段

- 引入 APS 有限产能排产
- 建立订单料工费成本核算
- 增加 AI 辅助排产、延期预警、瓶颈识别
