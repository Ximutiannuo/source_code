# 工程量日报：实战战果

工程量日报记录的是**昨天的战绩**，是进度核算与产值支付的唯一依据。准确、详尽地填报工程量完成量，不仅是日常管理的需要，更是积累公司「组织过程资产」与「经验数据库」的核心举措。这些底层数据将用于优化劳动定额、提升施工生产力，并为后续项目的成本核算与进度预测提供科学依据。

---

## 一、责任人：由谁来填报工程量日报

### 1.1 分包商计划管理员

* 负责**本单位（分包商）**范围内的工程量日报填报。
* 有权录入、修改本单位昨日实际完成的工程量数据。
* 不能操作其他分包商的数据。

### 1.2 子项目 / 计划部管理人员

* 负责**本子项目**或**全项目**范围内的日报管理。
* 有权查看、修改该范围内所有分包商填报的工程量日报。
* 可对分包商填报内容进行审核、修正或删除。

### 1.3 施工部核查责任

施工部有责任和义务检查分包商的工程量填报。如发现填报异常（如虚报、漏报、严重失实等），请及时反馈至**计划管理部**。计划管理部将对该分包商进行追责问责，并根据情节处以相应处罚，**根据合同约定最高可驱逐出场**。

---

## 二、填报要求

| 项目 | 说明 |
|------|------|
| **填报人** | 分包商计划管理员 |
| **截止时间** | **每日 12:00 前**（逾期将影响全项目的工效与进度核算） |
| **填报对象** | **昨天（前一日）**实际完成的工程量 |
| **页面入口** | 菜单 → 日报填报 → **工程量日报** |

---

## 三、核心流程：从选作业到保存

### 3.1 页面入口

菜单 → 日报填报 → **工程量日报**

### 3.2 作业的含义

工程量日报与**作业**一一关联。每条作业代表：

> **哪个分包商，在哪套图纸下，执行哪一个工作包的工作。**

* **分包商**：即施工承包商（如 C01、C02）。
* **图纸**：通过作业的子项、装置、主项等维度定位。
* **工作包**：即工程量类型（如焊接、混凝土等）。工作包定义可在 菜单 → **工作包管理** 中查阅。

**重要**：分包商计划管理人员在报量前，务必仔细核对**工作包是否准确无误**、**图纸编号是否准确无误**。工作包或图纸错误将导致进度核算、工效分析及产值确认出现偏差。

### 3.3 筛选作业并填报

1. 在工程量日报页面，通过顶部**筛选器**（分包商、子项目、主项、子项等）筛选出需要填报的作业。
2. 表格中每行对应一条**作业**或一条**工作步骤**；右侧为**昨日完成量**填报列，**人力**、**机械**为同日参考的信息提示，无需填报。
3. **昨日完成量**：在**工作步骤行**（位于作业行下方的缩进行）中输入完成量；作业行本身不填写完成量。
4. 直接在表格单元格中输入数值，无需弹出表单。
5. 填写完成后，点击**保存**按钮提交数据。

### 3.4 页面操作说明

#### 筛选器

页面顶部的**筛选器**（分包商、子项目、主项、子项、装置、区块等）用于缩小数据范围。按层级依次选择：子项目 → 开车阶段 → 装置 → 区块 → 主项 → 子项 等，可快速定位到您负责的区域。建议分包商用户先选定本单位（分包商）。

#### 分组、栏位、视图管理

* **分组**：点击「分组」按钮，选择分组字段（如专业、工作包、子项、分包商等），表格将按所选字段逐层分组；分组行可折叠/展开。
* **栏位设置**：点击「栏位设置」可勾选/取消需要显示的列。
* **视图管理**：可将当前分组方式、栏位配置、列宽保存为命名视图，下次可快速加载。

#### 导入与导出

| 操作 | 说明 |
|------|------|
| **导出 Excel** | 点击「导出 Excel」可导出当前筛选结果；**导出时需在筛选器中指定单一分包商**，以生成该单位的专属报表。 |
| **导入回写** | 点击「导入回写」，选择与系统导出一致格式的 Excel 文件，可将完成量批量导入；**完成量必须填写在「工作步骤」行**（描述列应对齐工作步骤名称），请勿在作业代码行填完成量；导入前请确保格式与导出一致。 |

### 3.5 需要填写的字段

| 字段 | 说明 | 是否必填 |
|------|------|----------|
| **昨日完成量** | 该作业下各工作步骤昨日实际完成的工程量；单位与工作包一致 | 选填 |
| 人力、机械 | 与人力日报一致，用于同日参考；工程量日报模式下可选填 | 选填 |

## 四、关键工程量（工作步骤）

每个工作包下可配置若干**工作步骤**，用于细分工程量类型。**关键工程量**参与进度核算与赢得值计算（如焊接米数、混凝土方量等），在表格中通常以绿色底纹标识。工作步骤定义由计划部在 **工作步骤管理** 中维护；若某工作包无工作步骤，则该作业**无需报量**。

**工艺管道 PI04/PI05**：该工作包数据由系统自动从诺德焊接数据库抓取，**无需分包商填报**。

---

## 五、工效与进度

* 工程量日报与人力日报共同支撑**工效**计算：工效 = 产出（工程量）÷ 人力投入。您可随时在**工效分析**模块查看施工效率是否达到标准定额。
* 工程量完成量参与**赢得值**与**进度**核算，是项目进度曲线和产值确认的基础数据。

---

## 六、注意事项

* 系统已按权限限制可见数据，您只能看到并操作本单位（分包商）范围内的作业。
* 数据应尽可能真实地反映每个工作步骤下的实际完成量。
* 导出报表时，需在筛选器中指定**单一分包商**，以生成该单位的专属报表。
* 施工管理部与计划部会对填报的工程量进行日常管理与核查，确保数据真实。
