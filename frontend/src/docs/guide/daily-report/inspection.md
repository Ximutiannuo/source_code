# 验收日报：质量闭环

验收日报由 **质量部 (QAQC)** 负责，确保施工完成与质量验收在系统内闭环。本文档说明业务流程、责任人、核心操作步骤及数据逻辑。

---

## 一、验收程序使用说明（ITP 查阅与导出）

验收程序页面用于查阅、筛选及导出 ITP（检验试验计划）及验收检查项，是填报验收日报时选择「ITP 文档」和「验收依据」的重要参考。

### 1.1 页面入口与权限

| 项目 | 说明 |
|------|------|
| **入口** | 菜单 → 系统管理 → **验收程序** |
| **权限** | 需要 `acceptance_procedure:read` 权限；无权限时页面提示请联系管理员分配 |

### 1.2 页面结构：ITP 主表

主表列出全部 ITP 文档，每行对应一条 ITP。列说明如下：

| 列名 | 说明 |
|------|------|
| **ITP 文档编号** | ITP 文档唯一编号，用于筛选和填报验收日报时选择 |
| **ITP 名称** | 文档名称 |
| **版本** | ITP 版本号 |
| **状态** | `active`（有效）、`draft`（草稿），可筛选 |
| **明细条数** | 该 ITP 下检查项（验收依据）的数量 |

### 1.3 展开查看验收明细（检查项）

* 点击行首 **>** 可展开该 ITP，查看其下的验收检查项。
* 明细表列说明：

| 列名 | 说明 |
|------|------|
| **No.** | 检查项编号（行号），对应验收日报中的「ITP 文档对应编号（验收依据）」 |
| **Work Description (Eng/Rus)** | 工作描述（英文/俄文） |
| **Applicable Documents** | 适用文件 |
| **Acceptance Criteria** | 验收标准 |
| **Quality Control Form** | 质控表 |
| **Sub / Con / Cust / AQC** | 分包商/承包商/业主/AQC 参与标识 |

* 填报验收日报时，选择的「ITP 文档对应编号」即为该 No. 列的值（如 1.1、3.2）。

### 1.4 筛选与导出

| 操作 | 说明 |
|------|------|
| **ITP 文档编号** | 在搜索框输入文档编号（支持部分匹配），点击搜索或回车 |
| **全部状态 / 仅有效** | 筛选全部 ITP 或仅 `active` 有效 ITP |
| **清除筛选** | 清空文档编号和状态筛选条件 |
| **刷新** | 重新加载列表 |
| **导出 Excel** | 按当前筛选条件导出 ITP 及明细到 Excel，便于离线查阅或打印 |

### 1.5 与验收日报的配合

* 验收日报新增记录时，「ITP 文档」下拉选择的值来源于此处维护的 ITP。
* 「ITP 文档对应编号（验收依据）」对应此处明细表的 **No.** 列。
* 若 ITP 列表为空或下拉框中无可选 ITP，说明尚未导入 ITP，请联系项目管理员从 Word 文档导入后再使用。

## 二、责任人：由谁来录入质量验收 RFI 日报

### 2.1 分包商 QAQC 工程师

* 负责**本单位（分包商）**范围内的 RFI 验收日报。
* 有权录入、修改、删除本单位填报的验收记录。
* 不能操作其他分包商的验收数据。

### 2.2 子项目 QAQC 主管

* 负责**本子项目**（如 ECU、PEL、UIO 等）范围内的质量验收。
* 有权录入本子项目的 RFI 日报；
* 有权查看、修改、审核**该子项目下所有分包商**填报的验收日报，可对分包商填报内容进行评价、修正或删除。
* 一个子项目下通常包含多个分包商，因此子项目 QAQC 主管具有跨分包商的审核权限。

### 2.3 其他角色

* **施工管理员**：只能查看验收日报，不能录入或修改。
* **系统管理员 / 计划经理**：可操作全部验收日报。

## 三、核心流程：从选作业到保存验收记录

### 3.1 页面入口

菜单 → 日报填报 → **验收日报**

### 3.2 作业的含义

验收日报与**作业**一一关联。每条作业代表：

> **哪个分包商，在哪套图纸下，执行哪一个工作包的工作。**

* **分包商**：即施工承包商（如 C01、C02）。
* **图纸**：通过作业的子项、装置、主项等维度定位；可手动填写「图纸编号」进一步关联。
* **工作包**：即工程量类型（如 ADD3、焊接、混凝土等）。工作包定义可在 菜单 → **工作包管理** 中查阅。

### 3.3 选中作业并点击「添加」

1. 在验收日报页面，通过顶部**筛选器**（分包商、子项目、主项、子项等）筛选出需要填报验收的作业。
2. 表格中每行对应一条**作业**，右侧有「验收」列。
3. 点击某作业行的「**+**」按钮（增加一条验收记录），弹出「新增验收日报」表单。
4. 若该作业已有验收记录，可点击「**≡**」按钮查看本作业的 RFI 清单，再选择编辑或新增。

### 3.4 点击「添加」后需要填写的字段

#### 基本信息

| 字段 | 说明 | 是否必填 |
|------|------|----------|
| **RFI 编号** | 如 GCC-CC7-UIO-ADD3-WS-RFI-99237 | 必填 |
| 短编码 | 自动取自 RFI 编号最后一段 | 自动 |
| 申请检查日期、计划验收日期 | 日期选择 | 选填 |
| 分包商、子项目、主项、子项、专业、工作包等 | 从作业带入，只读 | 自动 |
| 作业描述 | 来自作业或手动填写 | 选填 |
| 图纸编号 | 可输入多个，回车或逗号分隔 | 选填 |
| RFI/验收描述、验收地点 | 文本 | 选填 |

#### 验收信息

| 字段 | 说明 | 是否必填 |
|------|------|----------|
| **ITP 文档** | 下拉选择 ITP（编号 · 版本 · 名称） | 选填；选后才有验收依据可选 |
| **ITP 文档对应编号（验收依据）** | 即检查项行号，如 1.1、3.2 | 选填 |
| 验收类型 | 如 ACC | 选填 |
| 质检员、申请编号 | 文本 | 选填 |
| 数量 | 该 RFI 对应的实物量；单位应与工作包一致 | 选填 |

#### RFI 附件

* 填写**分包商**和 **RFI 编号**后，可上传：
  - **INPUT**：申请文件
  - **OUTPUT**：审批后文件
* 质检给出「拒绝」后，系统自动升版，下次上传 INPUT 为下一版本号，无需手动编号。

#### 结论与备注

* 验收结论、评论、问题整改说明、备注、验收通过日期等。

### 3.5 如何选择 ITP

1. 在「**ITP 文档**」下拉框中，选择对应的 ITP，格式为「文档编号 · 版本 · 名称」。
2. 若列表为空，说明尚未导入 ITP，请联系管理员。

### 3.6 如何选择 ITP 对应的编号（验收依据）

1. **先选择 ITP 文档**：选择某条 ITP 后，系统会加载该 ITP 下的检查项。
2. 在「**ITP 文档对应编号（验收依据）**」下拉框中，会显示该 ITP 下的检查项，格式为「行号 - 描述」（如 1.1 - 某项检查）。
3. 该编号用于区分关键/非关键验收项，参与后续工程量汇总（当前仅存储，尚未参与自动汇总）。

### 3.7 自动版本管理

* 质检拒绝后升版，系统自动管理 INPUT 版本号，无需手动编号。
* 分包商文件夹名由系统按分包商自动生成。

## 四、数据联动说明

### 4.1 验收日报与工程量控制清单（当前不联动）

**重要说明**：当前系统**不会**根据 RFI 验收日报自动更新工程量控制清单中的验收数据。

* **原因**：RFI 对应的验收数量尚未形成稳定的填报规范，联动条件暂不成熟。
* **现状**：工程量控制清单中的验收数据仍按原有方式（手工或既有流程）维护，不读取验收日报的汇总结果。
* **后续计划**：待数据录入情况改善、规则明确后，将考虑由系统自动汇总并更新，届时会另行通知。

### 4.2 关键工程量与验收依据编号的关系

* 每个工作包可配置若干个「关键验收项」（A/B/C 三个槽位），对应 ITP 中的检查项编号（如 4.1、4.2）。
* 填报验收日报时，若选择的「ITP 文档对应编号」与工作包配置的关键验收项一致，该条记录会被标记为参与关键工程量汇总（当前仅存储，尚未参与自动汇总）。
* 未匹配到关键验收项的记录，仍可正常保存数量，供后续使用。

## 五、注意事项

* 分包商文件夹名由系统按分包商自动生成。
* 请确保验收数据与分包商填报的工程量数据逻辑一致（如单位、工作包对应关系）。
* ITP 的详细结构可在「验收程序」页面查看，工作包定义可在「工作包管理」页面查看。
