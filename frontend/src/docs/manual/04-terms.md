# 术语定义与数据筛选指南

准确理解术语是进行数据分析的前提。本系统字段与国际通用标准及 P6 逻辑对齐。本文档说明**核心术语**、**空间层级**、**按模块查找定义表**及**自动计算逻辑**。

---

## 一、核心术语定义

| 术语（中文） | 英文/列名 | 说明 | 取值示例 |
|--------------|-----------|------|----------|
| **分包商** | Scope | 施工分包单位；权限及数据汇总的首要维度 | C01、C02、C05、C07、C09、C12、C13 等 |
| **子项目** | SubProject | 大的管理分区或项目部，按区域管控进度 | ECU、PEL、UIO |
| **主项** | Main Block | 主项编号，数字形态 | 12200、12401 |
| **子项** | Block | 子项，与主项层级嵌套 | 以系统为准 |
| **装置** | Unit | 装置代码 | EC1、EC2、PE1、PE2 等 |
| **开车阶段** | Train | 开车阶段 | T0、T1、T2 |
| **区块** | Quarter | 区块，取值形态为 Q 开头 | Q11、Q21 |
| **工作包** | Work Package | 工程量类型，决定计量标准 | 焊接、混凝土、防腐、钢结构等 |
| **计量单位** | UOM | 与工作包挂钩的法定计量单位 | m³、Tons、Dia-inch 等 |
| **专业** | Discipline | 与 work_package 相关，用于归类 | 与 rsc_defines 对应 |
| **实施阶段** | Implement Phase | 设计/采购/施工 | EN/设计、PR/采购、CT/施工 |
| **合同阶段** | Contract Phase | 合同阶段 | Add.1、Add.2.1、Add.2.2、Add.3 |

## 二、空间层级嵌套关系

筛选器采用**层级过滤**逻辑，请按以下顺序选择，确保路径正确：

```
子项目 (SubProject) → 开车阶段 → 装置 (Unit) → 区块 → 主项 (Main Block) → 子项 (Block)
```

* 若选了主项 A，却在子项里找主项 B 的作业，将找不到。
* 级联筛选选项来自 **facility_filter API** 及 **主项清单**，取值以系统实际数据为准。

## 五、自动计算逻辑

* **自动刷新**：页面数据定时自动同步，无需频繁手动刷新。
* **工效核算**：系统根据填报的人力与工程量，实时推算当日工效，自动生成趋势曲线，提前预警异常。
* **赢得值计算**：系统根据实际完成量自动计算赢得权重与实际人工时，详见《计算逻辑说明》。
