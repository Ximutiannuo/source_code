# 权限自查：为什么我看不见数据？

如果进入页面后是空白的，筛选器里点不出东西，或操作按钮不可用，请按以下顺序排查。**约 90% 的问题**源于筛选器或范围限制，而非权限不足。

---

## 一、第一步：检查筛选器（Filter）

### 1.1 Scope 选了吗？

* **分包商账号**：必须显式选择自己的 **Scope（分包商）** 才能看到作业。
* 若未选 Scope，作业清单、日报、专项计划等可能为空。
* 操作：顶部筛选器 → 选择您的分包商（如 C01、C02）。

### 1.2 层级选对了吗？

* 筛选器采用**层级过滤**：子项目 → 开车阶段 → 装置 → 区块 → 主项 → 子项 等。
* 若选了主项 A，却在子项里找主项 B 的作业，将找不到。
* 建议：点击「重置筛选器」后，按正确顺序重新选择。

### 1.3 筛选器与数据源一致吗？

* 如果某个作业在 **P6** 或 **作业汇总（Activity Summary）** 里没有分配给对应的分包商，系统中将无法找到该作业。
* 作业的新增、删除、分配均在 P6 中操作，同步到本系统约需 5 分钟。

## 二、第二步：检查角色与权限范围

### 2.1 分包商账号（Scope 受限）

| 能做的 | 不能做的 |
|--------|----------|
| 查看所属 Scope 的作业、计划、日报 | 查看其他分包商的数据 |
| 填报本单位人力日报、工程量日报 | 修改总体计划（确认完成、批量关闭等） |
| 编制、修改本单位专项计划（25 号前） | 25 号后修改专项计划（仅管理员可改） |
| 发起 HMD 问题、参与意见交流 | 审核/批准专项计划（需施工主管、计划经理等） |

### 2.2 QAQC 账号（验收权限）

| 能做的 | 不能做的 / 可能受限 |
|--------|----------------------|
| 填报验收日报（本单位或本子项目） | 可能无法修改施工工程量（视角色配置） |
| 查阅 ITP、验收程序 | 不能修改总体计划 |
| 审核本单位/本子项目下分包商的验收数据 | 部分操作受 scope/subproject 限制 |

### 2.3 计划部 / 计划经理 / 系统管理员

| 能做的 |
|--------|
| 全范围查看与维护计划、日报、工程量 |
| 总体计划：确认完成、重新打开、批量关闭 |
| 专项计划：审核、批准、撤回；25 号后仍可修改（仅系统管理员） |
| 工程量清单、主项清单、工作包、P6 同步等管理功能 |

### 2.4 施工主管（Construction Supervisor）

| 能做的 | 限制 |
|--------|------|
| 专项计划审核、批准、撤回 | **范围受子项目（subproject）约束**，如 ECUConstructionSupervisor 仅能操作 ECU 子项目 |

### 2.5 锁定状态：25 号封账

* **每月 26 日 00:00** 起，专项计划进入封账期。
* **普通用户**：无法修改计划量、备注、作业描述；无法删除、从作业池加入、导入。
* **仅系统管理员** 可变更专项计划。
* 若 26 号及以后无法修改专项计划，这是**正常现象**，请联系管理员处理。

## 三、第三步：常见问题速查

| 现象 | 可能原因 | 处理建议 |
|------|----------|----------|
| 页面空白 | 未选 Scope 或筛选层级错误 | 选择 Scope，检查层级，必要时重置筛选器 |
| 筛选器无选项 | 无对应权限或数据源为空 | 确认账号有 `planning:read`；检查 P6/作业汇总是否有数据 |
| 「确认完成」按钮不可用 | 无 `daily_report:create` 或 Scope 不匹配 | 联系管理员分配权限或核对 Scope |
| 工程量列不可编辑 | 无 `construction_volume:create` 等 | 联系管理员分配对应 `*_volume:create` |
| 专项计划 26 号后无法修改 | 封账期，仅管理员可改 | 联系系统管理员 |
| 验收日报无法填报 | 无 `inspection_db:create` | 联系管理员分配验收权限 |
| 工作包管理无「新增」按钮 | 非系统管理员 | 工作包增删改仅超级管理员或 `role_system_admin` |
| 账号管理菜单不显示 | 非超级管理员 | 账号管理仅超级管理员可见 |

---

## 四、权限与范围（Scope / Subproject）

### 4.1 范围限制

* 日报、计划、工程量、验收等权限支持 **scope** 或 **subproject** 范围限制。
* **Scope**：分包商维度，如 C01、C02；分包商账号通常绑定 scope。
* **Subproject**：子项目维度，如 ECU、PEL、UIO；项目部账号（如子项目 QAQC 主管、施工主管）通常绑定 subproject。
* **范围为空**：表示全范围，可操作全项目数据。

### 4.2 谁可以分配权限

* **超级管理员**（`is_superuser = true`）：拥有所有权限，可进入账号管理。
* **账号管理**：仅超级管理员可见；可为用户分配角色、权限及 scope/subproject。
* 普通用户无法自行修改权限，需联系管理员。

---

## 五、需要帮助时

1. **筛选器**：先确认 Scope 与层级选择正确。
2. **权限**：若确认筛选无误仍无数据，联系**计划管理部**或**系统管理员**核对账号权限与范围。
3. **数据源**：若作业在 P6 中不存在或未分配给对应分包商，需在 P6 中调整后等待同步。
