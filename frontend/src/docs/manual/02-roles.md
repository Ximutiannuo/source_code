# 职责分工：角色与责任

为保障系统正常运行与数据准确，各参与方需严格履行职责。本文档按**角色/部门**说明核心职责、典型任务与硬性时限。

---

## 一、角色与责任总览

| 角色/部门 | 核心职责 | 典型任务 | 硬性时限 |
|-----------|----------|----------|----------|
| **计划管理部** | 系统守护者 | 计划维护、P6 同步、专项计划审批、确认完成、批量关闭 | — |
| **分包商计划管理员** | 日报与专项计划填报第一责任人 | 人力/工程量日报；专项计划编制；作业池领取、周期计划量分配 | 每日 12:00 前；25 号前 |
| **质量部 (QAQC)** | 验收数据闭环 | 验收日报、ITP/RFI 填报 | — |
| **施工管理部** | 工程量共建与校核 | 工程量清单维护；分包商填报校核；专项计划审核/批准（施工主管） | — |
| **计划经理** | 计划管理最高权限 | 与计划管理部一致；全范围操作 | — |
| **系统管理员** | 系统运维与特殊调整 | 账号/权限管理；25 号后专项计划变更；工作包、设施等配置 | — |
| **全员** | 问题协作 | HMD 问题发起、意见交流、问题追踪 | — |

## 二、计划管理部

### 2.1 核心职责

* 系统整体运维，指导分包商及各部门开展业务。
* 与 P6 计划深度集成，确保线上线下计划一致。

### 2.2 典型任务

| 任务 | 说明 |
|------|------|
| **计划参数维护** | Calendar、WBS、Activity Code 等在 P6 中维护，同步至本系统。 |
| **作业管理** | 新增、删除、调整作业均在 P6 操作；本系统约 5 分钟同步。 |
| **总体计划** | 确认完成、重新打开、批量关闭；维护作业完成状态与锁定。 |
| **专项计划审批** | 审核、批准分包商/施工主管提交的专项计划；批准后计划量锁定。 |
| **分包商异常** | 对虚报、漏报、严重失实等填报异常进行追责，情节严重者可驱逐出场。 |

### 2.3 权限范围

* 全项目计划、日报、工程量、验收等数据的查看与维护。
* 确认完成、批量关闭需 `daily_report:create`；其他计划操作需 `planning:read/update` 等。

## 三、分包商计划管理员

### 3.1 核心职责

* **日报填报第一责任人**：人力日报（MPDB）、工程量日报（VFACTDB）。
* **专项计划编制**：月滚动、三月滚动计划；周期计划量分配。

### 3.2 典型任务

| 任务 | 说明 | 时限 |
|------|------|------|
| **人力日报** | 填报今日人力与机械投入；按作业关联。 | **每日 12:00 前** |
| **工程量日报** | 填报昨日实际完成的工程量；按作业关联。 | **每日 12:00 前** |
| **专项计划编制** | 从作业池添加作业；分配各周周期计划量；填写备注与作业描述。 | **每月 25 号前** |
| **HMD 问题** | 遇图纸、材料、移交等外部制约时，发起 HMD 问题；参与意见交流。 | — |

### 3.3 硬性时限

* **每日 12:00 前**：完成昨日工程量、今日人力的日报填报；逾期影响全项目工效与进度核算。
* **每月 25 号前**：完成当月专项计划量分配；26 日 00:00 封账后，普通用户无法修改。

### 3.4 权限范围

* 仅能操作**所属 Scope（分包商）**的数据；不能修改其他分包商的数据。
* 不能修改总体计划；不能确认完成、批量关闭作业。

## 四、质量部 (QAQC)

### 4.1 核心职责

* 验收类数据闭环；确保施工完成与质量验收在系统内同步。

### 4.2 典型任务

| 任务 | 说明 |
|------|------|
| **验收日报** | 填报 RFI 验收数据；与 ITP 检查项关联。 |
| **分包商 QAQC** | 填报本单位（分包商）范围内的 RFI 日报；不能操作其他分包商。 |
| **子项目 QAQC 主管** | 填报本子项目（ECU/PEL/UIO 等）的 RFI 日报；可查看、修改、审核该子项目下所有分包商的验收日报。 |
| **ITP 查阅** | 菜单 → 验收程序，查看 ITP 文档与检查项；导出 Excel 便于离线查阅。 |

### 4.3 权限范围

* 验收日报：录入、修改、审核；跨分包商审核仅限子项目 QAQC 主管。
* 施工工程量：可能仅有查看权限，不能修改（视角色配置而定）。

---

## 五、施工管理部

### 5.1 核心职责

* 与计划管理部共同进行工程量信息的管理与维护。
* 校核分包商填报数据的真实性；发现异常及时反馈计划管理部。

### 5.2 典型任务

| 任务 | 说明 |
|------|------|
| **工程量清单** | 维护 Volume Control List 的准确性；施工部责任人、工程量信息等。 |
| **校核责任** | 检查分包商的人力、工程量填报；发现虚报、漏报、严重失实则反馈计划管理部。 |
| **专项计划** | **施工主管**可审核、批准、撤回专项计划；范围受子项目（subproject）约束。 |

### 5.3 施工主管（Construction Supervisor）

* 角色：施工主管（如 ECUConstructionSupervisor、PELConstructionSupervisor）。
* 权限：专项计划审核、批准、撤回；范围受其所属子项目约束。
* 编制：施工主管也可编制专项计划（与分包商计划管理员、计划部相同）。

### 5.4 追责机制

* 如发现分包商填报异常，施工部有责任反馈至**计划管理部**。
* 计划管理部将追责问责，根据合同约定最高可**驱逐出场**。

---

## 六、计划经理

* 拥有计划管理相关**最高权限**；与计划管理部职责一致，可全范围操作。
* 可审核、批准专项计划；可确认完成、批量关闭作业；可操作 P6 同步、设施、工作包等。

---

## 七、系统管理员

### 7.1 核心职责

* 系统运维；账号与权限管理；特殊时期调整。

### 7.2 典型任务

| 任务 | 说明 |
|------|------|
| **账号与权限** | 创建用户、分配角色、部门、权限范围。 |
| **25 号后专项计划** | 封账后仅系统管理员可修改计划量、备注、作业描述；可删除、加入、导入。 |
| **工作包与设施** | 工作包管理、主项清单等配置；需 `system:admin` 或 `role_system_admin`。 |
| **系统配置** | P6 同步、焊接数据、AI 导出等管理功能。 |

---

## 八、HMD 问题解决部门（支持方）

专项计划 HMD 问题发起时，可选择**解决部门**，由对口部门处理并给出解决方案。常见解决部门包括：

| 部门 | 典型问题类型 |
|------|--------------|
| 设计管理部 | 设计与技术问题（图纸、技术澄清） |
| 采购管理部 | 采购与材料问题 |
| 仓储管理部 | 仓储与设备问题 |
| 施工管理部 | 施工管理问题 |
| 质量管理部 | 质量管理问题 |
| 安全管理部 | 安全环保问题 |
| 计划管理部 | 审批与流程问题 |
| 其他 | 行政后勤、IT、合同、费用控制、人力资源、竣工资料等 |

---

## 九、全员协作

* **HMD 问题**：任何账号均可发起「需要解决的问题」，选择问题类型、解决部门、责任人。
* **意见交流**：专项计划意见列、问题回复中可 @提及，被提及者会收到通知。
* **问题追踪**：异常可第一时间推送给责任人，确保事事有回音。
